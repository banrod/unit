# Experiment README Checklist

Use this checklist to validate that a new prototype or experiment directory is ready for collaborators.

## Metadata
- [ ] Title reflects the experiment's purpose.
- [ ] Authors and contact handles are listed.
- [ ] Status flag set (idea, prototype, ready-for-review).

## Environment
- [ ] Enumerate prerequisites (runtime versions, datasets, environment variables).
- [ ] Document setup commands and teardown steps.
- [ ] Link to any helper scripts in `workstation/scripts/`.

## Validation
- [ ] Include step-by-step instructions for running tests or demos.
- [ ] Summarize expected outputs or acceptance criteria.
- [ ] Capture benchmarking or profiling notes when available.

## Knowledge Transfer
- [ ] Reference related notes or assessments for additional context.
- [ ] Highlight open questions or follow-up tasks.
- [ ] Provide guidance on how to promote the experiment into the main product surface.
