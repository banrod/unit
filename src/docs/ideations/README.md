# Ideations

This document collects potential directions and ideas for how the Unit
system may evolve. These items are not part of the formal roadmap but
serve as inspiration for new experiments and community discussion.

## Possible Enhancements

- **Mobile editing experience** – refine touch and gesture support to
  make creating graphs on phones and tablets practical.
- **Real‑time collaboration** – allow multiple users to modify a graph
  simultaneously via a shared session.
- **Extensible primitive library** – publish a process for users to
  create and share their own primitives and components.
- **Package export** – generate self‑contained bundles that can run
  offline or be embedded into existing sites and apps.
- **Machine learning units** – wrap common ML tools as primitives so
  visual programs can integrate basic models.
- **Remote system control** – orchestrate and monitor units running on
  distributed servers from a single editor.

These concepts highlight only a subset of what the Unit platform might
support in the future. Community feedback is welcome, and pull requests
exploring any of these ideas are encouraged.
